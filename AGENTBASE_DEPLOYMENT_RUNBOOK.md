## 5. AgentBase Deploy Direction

Recommended runtime type: Custom Agent.

Current public endpoint:

```text
https://endpoint-14358055-e169-46fc-9328-8e14537c46cd.agentbase-runtime.aiplatform.vngcloud.vn
```

Quick public smoke test:

```bash

  ./scripts/demo_curl.sh
```

Recommended deployment path for the competition demo:

1. Use AgentBase managed Container Registry.
2. Build Docker image for `linux/amd64`.
3. Push image to AgentBase CR.
4. Create/update Agent Runtime.
5. Pass `.env` as runtime env file.

Runtime recommendation:

- Runtime name: `be-hadi-bao-hiem`
- Network: `VPC` if the demo must read live Tableau/Atlas data.
- Network fallback: `PUBLIC` only if the runtime can resolve/reach `atlas.vng.com.vn`, or if the demo uses uploaded/sample CSV.
- Flavor: choose the smallest flavor that supports the selected network mode. Start with `1x1-general` for PUBLIC, or the equivalent `1x1` VPC flavor if available.
- Min replicas: `1`
- Max replicas: `1`

Recommended env file:

- `.env` generated by `scripts/setup_agentbase_credentials.sh`
- Add Tableau config with `scripts/import_tableau_mcp_env.sh`
- Add `TABLEAU_VIEW_PRODUCT_PERFORMANCE_ID` when there is a Tableau view that exports AppID/SKU/product rows.

Important: choose `VPC` for live Tableau if Atlas is internal/VPN-only. If the selected Tableau view only exports aggregate `All` rows, Bé Hadi will not invent AppID numbers; it needs a product-level view id or a CSV export that contains `App ID`, `SKU Name`, `Product Name`, `TPV`, and `MPU`.
